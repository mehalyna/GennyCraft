# GennyCraft MCP Server - Setup Guide

This guide explains how to set up and use the GennyCraft MCP (Model Context Protocol) server to interact with your GennyCraft backend from VS Code.

## Prerequisites

- Node.js installed
- Python 3.x installed
- GennyCraft Django backend running
- VS Code with GitHub Copilot extension

## What's New ✨

The MCP setup now uses an **automatic token management system**:

✅ **No manual editing of `.vscode/mcp.json`** — Configuration is done once  
✅ **Token auto-saved to file** — Run `get_jwt_token.py` and you're done  
✅ **Secure by default** — `.gennycraft-token` is in `.gitignore`  
✅ **Simple token refresh** — Just re-run the script when it expires  

**Old Flow (manual):**
```
Get token → Copy → Paste into .vscode/mcp.json → Reload VS Code
```

**New Flow (automatic):**
```
Run get_jwt_token.py → Reload VS Code
```

## Quick Start (First Time Setup)

### 1. Build the MCP Server

The GennyCraft MCP server is located at:
```
C:\Users\hmeln\source\repos\gennycraft-mcp
```

Navigate to the MCP server directory and build it:

```powershell
cd C:\Users\hmeln\source\repos\gennycraft-mcp
npm install
npm run build
```

### 2. Start the Django Backend

Make sure your Django backend is running:

```powershell
cd C:\Users\hmeln\source\repos\GennyCraft
python manage.py runserver
```

The server should be running at `http://127.0.0.1:8000/`

### 3. Generate Your First JWT Token

Run the token generator script:

```powershell
python get_jwt_token.py test@example.com testpass123
```

**What this does:**
- Creates a test user (if doesn't exist)
- Generates a JWT access token
- **Automatically saves it to `.gennycraft-token` file**
- The MCP server will read this file automatically

**Output:**
```
✓ Created user: test@example.com

============================================================
JWT ACCESS TOKEN:
============================================================
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
============================================================

✓ Token saved to .gennycraft-token
  MCP server will read from this file automatically
============================================================
```

### 4. Reload VS Code

Start using the MCP server:

1. Press `Ctrl+Shift+P`
2. Type "Developer: Reload Window"
3. Press Enter

**That's it!** The MCP server is now ready to use.

## How It Works

The GennyCraft MCP setup uses a wrapper script for seamless token management:

```
.vscode/mcp.json → gennycraft-mcp-wrapper.js → .gennycraft-token → MCP Server
```

1. **`.vscode/mcp.json`** - Points to the wrapper script (already configured)
2. **`gennycraft-mcp-wrapper.js`** - Reads token from file and starts MCP server
3. **`.gennycraft-token`** - Contains your JWT token (auto-generated by `get_jwt_token.py`)

**No manual configuration needed!** Just run `get_jwt_token.py` when your token expires.

## Using the MCP Server

### Available Tools

The GennyCraft MCP server provides the following tools:

#### 1. Get Monthly Transactions

Fetch all transactions for the authenticated user in a specific month.

```
@gennycraft get_monthly_transactions Month: 2026-04
```

**Parameters:**
- `month` (required) - Format: YYYY-MM
- `category` (optional) - Filter by category ID

**Example Response:**
```json
{
  "count": 2,
  "results": [
    {
      "id": 5,
      "type": "expense",
      "amount": "10.00",
      "currency": "USD",
      "date": "2026-04-02T11:24:00Z",
      "category_name": "Groceries",
      "category_icon": "🛒",
      "title": "Products"
    }
  ]
}
```

#### 2. Get Monthly Report

Generate a spending summary report for a month.

```
@gennycraft get_monthly_report Month: 2026-04
```

**Parameters:**
- `month` (required) - Format: YYYY-MM

Returns total income, expenses, and balance for the period.

**Example Response:**
```json
{
  "period": {
    "start": "2026-04-01T00:00:00",
    "end": "2026-04-30T23:59:59"
  },
  "income": {
    "total": 100.0,
    "count": 1
  },
  "expense": {
    "total": 10.0,
    "count": 1
  },
  "balance": 90.0
}
```

#### 3. Check Budget Status

⚠️ **Not yet implemented** - Budget functionality coming soon.

```
@gennycraft check_budget_status Category ID: 5
```

**Parameters:**
- `category_id` (required) - Category ID

Returns error message indicating feature not implemented.

#### 4. Get Account Balance

Get current balance for a specific account.

```
@gennycraft get_account_balance Account ID: 1
```

**Parameters:**
- `account_id` (required) - Account ID

**Example Response:**
```json

  "name": "Main Wallet",
  "balance": "1000.00",
  "currency": "USD",
  "account_type": "cash"
}
```

## Troubleshooting

### MCP Server Not Connecting

**Problem:** Cannot connect to gennycraft MCP server

**Solution:**
1. Verify the wrapper script exists: `gennycraft-mcp-wrapper.js`
2. Make sure the MCP server is built: `C:\Users\hmeln\source\repos\gennycraft-mcp\build\index.js`
3. Rebuild the MCP server:
   ```powershell
   cd C:\Users\hmeln\source\repos\gennycraft-mcp
   npm run build
   ```
4. Reload VS Code window

### Authentication Errors

**Problem:** `{"detail":"Authentication credentials were not provided."}`

**Solution:**
Generate a new JWT token (it auto-saves to `.gennycraft-token`):
```powershell
python get_jwt_token.py test@example.com testpass123
```

Then reload VS Code window (`Ctrl+Shift+P` → "Developer: Reload Window").

### Connection Refused

**Problem:** `fetch failed` or connection refused errors

**Solution:**
1. Make sure Django backend is running: `python manage.py runserver`
2. Verify the backend is accessible at `http://localhost:8000/api`
3. Check that the wrapper reads the correct API URL (defaults to `http://localhost:8000/api`)

### Token Expired

**Problem:** Authentication suddenly stops working after some time

**Solution:**
JWT tokens expire after 60 minutes. Simply regenerate:
```powershell
python get_jwt_token.py test@example.com testpass123
```

The token is automatically saved to `.gennycraft-token`. Just reload VS Code afterwards.

### Token File Missing

**Problem:** Error reading token file: `.gennycraft-token`

**Solution:**
You haven't generated a token yet. Run:
```powershell
python get_jwt_token.py test@example.com testpass123
```

## Configuration

### MCP Wrapper Configuration

The `gennycraft-mcp-wrapper.js` automatically configures:

| Variable | Description | Default |
|----------|-------------|---------|
| `GENNYCRAFT_API_URL` | Django backend API URL | `http://localhost:8000/api` |
| `GENNYCRAFT_JWT_TOKEN` | Read from `.gennycraft-token` file | Auto-loaded |

### Token File Location

The JWT token is stored in:
```
.gennycraft-token
```

This file is:
- Auto-generated by `get_jwt_token.py`
- Auto-read by `gennycraft-mcp-wrapper.js`
- Should be added to `.gitignore` (security best practice)

### Advanced: Custom API URL

To use a different API URL, edit `gennycraft-mcp-wrapper.js`:

```javascript
env: {
  ...process.env,
  GENNYCRAFT_API_URL: 'https://your-api-url.com/api',  // Change this
  GENNYCRAFT_JWT_TOKEN: token
}
```

## Security Notes

⚠️ **Important Security Considerations:**

1. **Never commit JWT tokens**
   - Add `.gennycraft-token` to `.gitignore`
   - The `.vscode/mcp.json` no longer contains tokens (safe to commit)

2. **Token file is auto-managed**
   - Tokens are stored in `.gennycraft-token`
   - This file should NOT be committed to version control
   - Regenerate tokens easily with `get_jwt_token.py`

3. **Token expiration**
   - Tokens expire after 60 minutes by default
   - Simply run `get_jwt_token.py` again to refresh

4. **Use different tokens for development and production**
   - Create users with different emails for different environments
   - Example: `dev@gennycraft.com`, `prod@gennycraft.com`

5. **Secure storage**
   - Tokens are stored locally in your project directory
   - Never share your `.gennycraft-token` file
